Install · Claude Code

Install Nio for Claude Code.

Download the platform zip, run setup.sh, and Nio registers a marketplace + plugin with Claude Code, installs PreToolUse / SessionStart hooks, and writes default config to ~/.nio/.

Requirements

  • Claude Code CLI installed and claude on PATH
  • Node.js ≥ 18 on PATH (the guard scripts are Node)
  • macOS, Linux, or WSL

Install

One-liner

VERSION=$(curl -s https://api.github.com/repos/core0-io/nio/releases/latest | grep tag_name | cut -d'"' -f4) && \
curl -LO "https://github.com/core0-io/nio/releases/download/${VERSION}/nio-claude-code-${VERSION}.zip" && \
unzip -o "nio-claude-code-${VERSION}.zip" -d nio-claude-code && \
cd nio-claude-code && ./setup.sh

Manual

  1. Grab nio-claude-code-v<version>.zip from the Releases page.
  2. unzip nio-claude-code-v<version>.zip -d nio-claude-code
  3. cd nio-claude-code && ./setup.sh

If you use both Claude Code and OpenClaw, grab nio-all-v<version>.zip instead — the bundled setup.sh auto-detects which agents you have and sets up both.

What setup.sh does

  1. Verifies Node.js ≥ 18 is on PATH.
  2. Registers the nio marketplace with Claude Code: claude plugin marketplace add <script-dir>
  3. Installs the plugin: claude plugin install nio@nio
  4. Syncs scripts + hooks into the Claude Code plugin cache (so re-running the script picks up local edits).
  5. Creates ~/.nio/ and writes config.yaml from the defaults (kept if it already exists, unless you pass --reset-config).

Flags

FlagPurpose
--cc-home <path>Override the Claude Code config dir (default $CLAUDE_CONFIG_DIR, else ~/.claude).
--reset-configOverwrite ~/.nio/config.yaml with the bundled defaults. Use after upgrades that changed the schema.
--uninstallRemove the plugin, marketplace entry, and ~/.nio/ directory.
-h, --helpPrint usage.

Where things live

PathWhat
~/.nio/config.yamlGuard + collector config. See Config reference.
~/.nio/audit.jsonlAppend-only JSONL audit log: guard decisions, scans, lifecycle events, and every hook event (PreToolUse, PostToolUse, TaskCreated, TaskCompleted, Stop, SubagentStop, …). Read via /nio report.
~/.nio/traces-state-store.jsonCross-process trace state (turn IDs + pending span starts). Internal; sits next to audit.jsonl.
~/.claude/plugins/…Plugin cache — managed by Claude Code; don't edit directly.

Set NIO_HOME to move the config/audit log elsewhere (e.g. NIO_HOME=~/Projects/nio-state), or set collector.logs.path in config.yaml to relocate just the audit log + state file.

Verify

  1. Open a fresh Claude Code session (hooks take effect on the next session, not the one in flight).
  2. In chat, type /nio config show — the current protection level and config path print.
  3. Try a scan: /nio scan .
  4. Make the agent run something benign (ls), then /nio report — the event is there.
If /nio isn't available

Re-register the marketplace manually:

claude plugin marketplace add "$(pwd)"    # inside nio-claude-code/
claude plugin install nio@nio

If the marketplace path is stale (e.g. you moved the unzip folder), setup.sh detects and re-registers it automatically on next run.

Upgrade

Download the new zip, run ./setup.sh. Existing config is preserved. If the new version changed the schema, add --reset-config.

Uninstall

./setup.sh --uninstall

Removes the plugin, marketplace entry, and ~/.nio/. Safe to re-run.
