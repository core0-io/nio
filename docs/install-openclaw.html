Install · OpenClaw

Install Nio for OpenClaw.

The OpenClaw plugin is a bundled single-file plugin.js. Run setup.sh to register it with the OpenClaw CLI and write the default config to ~/.nio/.

Requirements

  • OpenClaw (ClawHub) CLI installed and openclaw on PATH
  • Node.js ≥ 18 on PATH
  • macOS, Linux, or WSL

Install

One-liner

VERSION=$(curl -s https://api.github.com/repos/core0-io/nio/releases/latest | grep tag_name | cut -d'"' -f4) && \
curl -LO "https://github.com/core0-io/nio/releases/download/${VERSION}/nio-openclaw-${VERSION}.zip" && \
unzip -o "nio-openclaw-${VERSION}.zip" -d nio-openclaw && \
cd nio-openclaw && ./setup.sh

Manual

  1. Grab nio-openclaw-v<version>.zip from the Releases page.
  2. unzip nio-openclaw-v<version>.zip -d nio-openclaw
  3. cd nio-openclaw && ./setup.sh

What setup.sh does

  1. Verifies Node.js ≥ 18.
  2. Registers the plugin with OpenClaw (installs the bundled plugin/plugin.js).
  3. Mirrors the skill directory so /nio appears in the skill list.
  4. Creates ~/.nio/ and writes config.yaml from defaults (kept if present, unless --reset-config).

Flags

FlagPurpose
--openclaw-home <path>Override the OpenClaw state dir (default $OPENCLAW_STATE_DIR, else ~/.openclaw).
--reset-configOverwrite ~/.nio/config.yaml with bundled defaults.
--uninstallRemove the plugin and ~/.nio/.
-h, --helpPrint usage.

Where things live

PathWhat
~/.nio/config.yamlGuard + collector config. See Config reference.
~/.nio/audit.jsonlAppend-only JSONL log of guard decisions, scans, and lifecycle events.
~/.openclaw/plugins/nio/Installed plugin bundle. Managed by OpenClaw; don't edit directly.

Set NIO_HOME to move the Nio state directory (config/audit/metrics) elsewhere.

Confirm behavior on OpenClaw

No interactive prompt

OpenClaw doesn't expose an interactive "do you want to run this?" UI. When the guard decides confirm on OpenClaw, the confirm_action fallback kicks in. Default: allow (let through and log a warning). Options:

  • allow — let it through, record a warning in the audit log (default).
  • deny — block the action. Recommended for headless environments where you can't manually review.
  • ask — use platform confirm if available, else fall back to allow.

Set it via /nio config or edit ~/.nio/config.yaml directly. See Config reference → confirm_action.

Verify

  1. Open OpenClaw. The /nio skill should be in the skill list.
  2. Run /nio config show.
  3. Run /nio scan . on a small folder.
  4. Check /nio report to see the scan event.

Uninstall

./setup.sh --uninstall

Removes the plugin and ~/.nio/. Safe to re-run.
